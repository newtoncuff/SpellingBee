## 2. Introduction

### 2.1 Purpose
The Spelling Learner Web App is designed to help children improve spelling skills by interacting with image-based word puzzles. The app presents words of varying difficulty (3–10 letters) alongside a relevant image, prompting users to fill in missing letters using a selectable alphabet. The application is intended for educational use.

### 2.2 Scope
The app enables children to:
- Select a difficulty level (easy, medium, hard) based on word length.
- Solve spelling puzzles by selecting letters to fill blanks sequentially.
- View immediate visual feedback on their selections with color-coded responses.
- Track progress with a star-based reward system (10 consecutive correct answers).
- Avoid repetition of word/image combos within the last 10 tasks.
- Experience celebration fireworks upon achieving 10 consecutive correct answers.

The app also includes an administrative interface that allows:
- Adding new puzzles manually or through auto-upload functionality.
- Managing existing puzzles (edit/delete).
- Uploading images with automatic puzzle generation based on filename.

All data, including words, images, user progress, and task history, is stored in a SQLite database. The application is fully containerized using Docker for easy deployment.

### 2.3 Definitions, Acronyms, and Abbreviations
- **Task:** A single spelling puzzle.
- **Combo:** Word/image pairing.
- **Difficulty:** Word length grouping (easy: 3–5, medium: 5–7, hard: 7–10).
- **Blank:** A missing letter in a word.

## 4. Functional Requirements

### 4.1 Difficulty Selection
- Users can choose difficulty (easy: 3–5 letters, medium: 5–7, hard: 7–10).
- The app generates puzzles accordingly.

### 4.2 Puzzle Generation
- App selects a random word (matching difficulty) and its associated image.
- Randomly removes letters from the word for user to fill in.
- Ensures the same word/image combo is not shown within the previous 10 tasks.

### 4.3 Interactive Puzzle Solving
- Display the word with missing letters as blanks, with visible letters shown in regular format.
- Show a relevant image above the word loaded from the backend API.
- Alphabet displayed below: all letters in **white** with **colorful borders** (randomized colors), always selectable and unchanged on selection.
- Sequential letter filling system:
  - Only the **next blank in order** (left to right) is active for input.
  - User must complete blanks sequentially - cannot skip ahead.
  - When a user clicks a letter:
    - The letter fills the **current active blank**.
    - If correct:
      - Letter turns **green** with white text.
      - A **green check mark (✓)** appears above the letter.
      - System automatically advances to the next blank.
      - When all blanks are filled correctly, word is automatically submitted.
    - If incorrect:
      - Letter turns **red** with white text.
      - Blank is cleared after **2 seconds**.
      - User remains on the same blank position.
- Letters in the alphabet can be selected multiple times (to fill repeated letters in words).
- Submit button appears and becomes active only when all blanks are completed.
- Immediate submission occurs when the final blank is filled correctly.

### 4.4 User Progress Tracking & Reward System
- **Star-based Progress Display:** Shows 10 star positions with filled/unfilled states.
- **Consecutive Correct Tracking:** Tracks consecutive correct answers across puzzle sessions.
- **Celebration System:** When user reaches 10 consecutive correct answers:
  - Full-screen fireworks animation is displayed.
  - Congratulatory message appears with celebration text.
  - Progress resets to 0 after celebration.
- **Visual Progress Indicators:** Stars fill with golden images as progress increases.
- **Persistent Progress:** Progress is maintained across browser sessions via database storage.

### 4.5 Data Management & Database Schema
- **SQLite Database** with automatic initialization and schema creation:
  - `words` table: id, text, difficulty
  - `images` table: id, file_path, description  
  - `combos` table: id, word_id, image_id (links words to images)
  - `tasks` table: id, user_id, combo_id, date, completed, correct
  - `users` table: id, name, consecutive_correct
- **Automatic Data Population:** Database auto-populates from available image files on startup.
- **Image Management:** Automatic word extraction from image filenames (e.g., "cat.jpg" → "CAT" puzzle).
- **Difficulty Auto-Assignment:** Word difficulty automatically determined by character length.

### 4.6 Administrative Interface
- **Admin Panel Access:** Available at `/admin` route with full CRUD operations.
- **Quick Upload Feature:** Drag-and-drop or click-to-upload with automatic puzzle creation.
- **Manual Puzzle Creation:** Form-based interface for creating custom word/image combinations.
- **Puzzle Management:** Edit existing puzzles, delete puzzles, view all puzzles in table format.
- **Image Upload & Management:** Upload new images with automatic filename-to-word conversion.
- **Visual Feedback:** Real-time image availability checking and thumbnail previews.
- **Bulk Operations:** Support for multiple puzzle operations and batch management.

  - Custom `SpellingBeeDatabase` class with automatic initialization
  - Comprehensive CRUD operations for all entities
  - Anti-repetition logic for puzzle selection
  - Progress tracking and statistics calculation

## 7. Data Design

### 7.1 Database Schema (IMPLEMENTED)
- **words:** 
  - `id` (INTEGER PRIMARY KEY AUTOINCREMENT)
  - `text` (TEXT NOT NULL) - The word to be spelled
  - `difficulty` (TEXT NOT NULL) - 'easy', 'medium', or 'hard'

- **images:** 
  - `id` (INTEGER PRIMARY KEY AUTOINCREMENT)
  - `file_path` (TEXT NOT NULL) - Filename of the image
  - `description` (TEXT) - Alt text description

- **combos:** 
  - `id` (INTEGER PRIMARY KEY AUTOINCREMENT)
  - `word_id` (INTEGER NOT NULL) - Foreign key to words table
  - `image_id` (INTEGER NOT NULL) - Foreign key to images table
  - `UNIQUE(word_id, image_id)` - Prevents duplicate combinations

- **tasks:** 
  - `id` (INTEGER PRIMARY KEY AUTOINCREMENT)
  - `user_id` (INTEGER) - References users table
  - `combo_id` (INTEGER NOT NULL) - Foreign key to combos table
  - `date` (TEXT NOT NULL) - ISO timestamp of task creation
  - `completed` (BOOLEAN NOT NULL DEFAULT 0) - Task completion status
  - `correct` (BOOLEAN NOT NULL DEFAULT 0) - Answer correctness

- **users:** 
  - `id` (INTEGER PRIMARY KEY AUTOINCREMENT)
  - `name` (TEXT) - User identifier
  - `consecutive_correct` (INTEGER DEFAULT 0) - Progress tracking for rewards

### 7.2 Data Validation
- Words: 3–10 letters; images linked to words; no duplicate combos.

```dockerfile
COPY package.json package-lock.json ./
RUN npm ci

COPY . .

EXPOSE 3000

# Serve production build with a simple static server
RUN npm install -g serve
CMD ["serve", "-s", "build", "-l", "3000"]
```

#### Production Docker Compose (`docker-compose.yml`)
```yaml
version: '3.9'

services:
  backend:
    build: ./backend
    container_name: spelling_backend
    ports:
      - "5000:5000"
    volumes:
      - ./data:/app/data
    environment:
      - FLASK_ENV=production
      - PYTHONUNBUFFERED=1

    container_name: spelling_frontend
    ports:
      - "3000:3000"
    depends_on:

    environment:
```
